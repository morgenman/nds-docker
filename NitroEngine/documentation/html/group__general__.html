◆ NE_GPUIsRendering()

bool NE_GPUIsRendering ( void  )

Returns false if VCOUNT is between 192 and 213. If not, true.

If true, you should't load textures (during the game). If you try to load textures, there is a moment when the GPU can't access that data, so there will be glitches.

◆ NE_Viewport()

void NE_Viewport ( int  x1,
int  y1,
int  x2,
int  y2 
)

Sets the part of screen which displays the 3D image and sets a new projection.

Parameters
x1(x1, y1) Down-left pixel.
y1(x1, y1) Down-left pixel.
x2(x2, y2) Up-right pixel.
y2(x2, y2) Up-right pixel.

In dual 3D mode it is reseted in NE_ProcessDual(), so you have to call it inside your drawing function.

After NE_2DViewInit() viewport is set to (0,0,255,191) for a while, in the next NE_Process() it will go back to what you set (in Dual 3D it goes to default values).

You can't set another viewport for 2D objects with this function as it changes the projection matrix, use glViewport(). This isn't saved anywhere, you will have to use it after NE_2DViewInit() every frame.

◆ NE_WaitForVBL()

void NE_WaitForVBL ( NE_UpdateFlags  flags)

Updates selected systems and waits for vertical blank.

Parameters
flagsLook at NE_UpdateFlags.

For example, NE_WaitForVBL(NE_UPDATE_GUI | NE_UPDATE_ANIMATIONS);

NE_CAN_SKIP_VBL allows the function skip the VBL wait if CPU load is greater than 100. You should use this if you don't need to load textures or do anything else during VBL. You have to set NE_HBLFunc() to HBL interrupt function for this to work.
